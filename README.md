# 🏦 Bank Management System  
### C++ Console Application | OOP | File Handling

---

## 📌 Overview
A console-based banking system developed using **C++ and Object-Oriented Programming (OOP)**.  
The system simulates real banking operations with a clean architecture, role-based permissions, and persistent file storage.

It focuses on real-world system design with clear separation between:
- Business Logic  
- User Interface  
- Data Management  

---

## ⚙️ Features

### 👤 User & Permissions System
- Multiple user roles (Admin / Employee / Client)
- Role-based access control
- Secure authorization for operations

---

### 🏦 Bank Operations
- Create / Update / Delete Clients  
- Manage user accounts  
- Deposit & Withdraw funds  
- Transfer money between accounts  
- Currency conversion system  

---

### 💾 Data Persistence
- File-based storage system
- Each entity has its own dedicated file:
  - Users.txt  
  - Clients.txt  
  - Accounts.txt  
  - Transactions.txt  
- Automatic loading and saving of data

---

### 🧠 Validation & Security
- Input validation for all operations
- Error handling for invalid transactions
- Protection against unauthorized access

---

## 🏗️ System Architecture

The system follows a layered architecture:

Business Logic Layer  
→ Core banking operations  

UI Layer (Console Interface)  
→ Handles user interaction  

Data Layer (File System)  
→ Responsible for file storage and retrieval  

### ✔️ Design Principles Used
- Encapsulation  
- Abstraction  
- Separation of Concerns  
- Modular Design  

---

## 📁 Data Structure

Each module is stored separately:

Users.txt  
Clients.txt  
Accounts.txt  
Transactions.txt
